OOPs

Objects can be defined using the following object creation patterns.

Objct Literal Pattern

The object literal pattern is only used to define simple objects. You can test the code below by typing into the browser address bar, javascript:alert(objectLiteral.firstName);, or use your browsers debugging tools.


  // Object literal pattern
  objectLiteral = {
    firstName: 'Geraint',
    lastName: 'Anderson',
  };

Object Constructor Pattern

A better method is to create an object constructor function. This is just a function that helps build the object, like a class in other languages.


  // Object Constructor pattern
  function ObjectConstructor(firstName, lastName) {
    this.firstName = firstName;
    this.lastName = lastName;
    this.fullName = function(){
      return this.firstName + ' ' + this.lastName;
    };
  };

New objects can then be created using the "new" keyword.


  var me = new ObjectConstructor('Geraint', 'Anderson');
  alert(me.fullName());

Object Prototype Pattern

The object prototype pattern is used to inherit properties and functions. In practice, the object prototype pattern and the object constructor pattern are most commonly used, usually together.


  // An ordinary object constructor is created
  function prototypePattern(){
    this.setName = function(firstName, lastName){
      this.firstName = firstName;
      this.lastName = lastName;
    };
  };
  // It inherits from our previous object
  prototypePattern.prototype = new ObjectConstructor();
  // We create an instance, which has the original and new properties and functions
  var ger = new prototypePattern();
  ger.setName('Adam','Adams');
  ger.fullName();

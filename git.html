GitHub

Terminology

Repository

BranchingA way to work on multiple parts of the reporitory at once. New branches should be created when working on each new feature/bugfix.

CommitsSaved changes

IssueA note on a repository about something that needs attention

Pull RequestPropose the changes should be added to the master branch. Someone needs to pull the changes in following a pull request.

Actions

Change a repository

  1. In the repository, click the branch dropdown.
  2. In the box,type a new repository name and press enter.
  3. Modify the required files.
  4. Commit the changes.
  5. Go to the pull requests page from the sidebar.
  6. Click "New Pull Request"
  7. In the dropdown choose the branches you with to merge. Master is the main one and compare is the chaged branch.
  8. Compare the changes then click "Create Pull Request".
  9. Add Comments. If this is fixing an issue, in the title put "fixed #" where # is the issue number.
  10. click "Create Pull Request"
  11. Click "Merge pull request"
